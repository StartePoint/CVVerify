# Support

Use this document to decide where to ask questions and what information to provide when you need help with CVVerify.

## What Support Covers

Support requests are appropriate for:

- build or environment setup problems
- questions about repository structure
- operator workflow confusion
- uncertainty about which issue template or contribution path to use
- clarification on bundled dependency expectations

## Where To Start

Before opening an issue, check:

- `README.md`
- `docs/build-windows-mingw.md`
- `docs/repository-structure.md`
- `CONTRIBUTING.md`
- existing issues and pull requests

If the problem turns out to be a reproducible bug or a clear missing feature, open a GitHub issue with the matching template instead of a generic support request.

## What To Include

When asking for help, include:

- operating system
- Qt version
- compiler and toolchain
- whether you are using CMake or qmake
- whether you are using the bundled OpenCV SDK
- the command you ran
- the exact error message
- screenshots when the problem is visible in the UI

For operator or preview questions, also include:

- the input media type
- the operator or pipeline involved
- the parameter values that matter

## Security Reports

Do not use support channels for security-sensitive reports. Follow `SECURITY.md` instead.

## Response Expectations

This project is still pre-release. Support is best-effort and depends on maintainer availability.

Clear, minimal reproductions are much easier to help with than broad problem descriptions.
