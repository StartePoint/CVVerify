# Security Policy

## Supported Versions

This project is currently pre-release. Security fixes will be handled on the main development line until stable versioning is established.

## Reporting a Vulnerability

Please do not open public issues for security-sensitive reports.

Instead, report vulnerabilities privately to the project maintainer through a secure contact channel once published.

When reporting, include:

- affected version or commit
- reproduction steps
- impact summary
- any suggested mitigation

We will acknowledge reports as quickly as practical and coordinate responsible disclosure.
