CVVerify Operator System Review

算子参数审阅页

这是给你确认的一版“算子目录 + 参数规范”提案，不是最终代码实现清单。 我把 9 大类算子拆成了可以直接挂进现有 `OperatorDescriptor + StepSchema` 架构的形式， 同时标出了哪些算子需要画布交互、第二张图输入、SURF 占位、以及哪些视觉定义仍待你拍板。

这页的阅读方式

  • 先看每个算子的“建议实现方式”：独立算子还是家族算子。
  • 再看参数表，确认默认值、范围、是否需要条件显示。
  • 重点帮我拍板 4 类问题：`光纤效果`、`特殊效果`、交互型参数、SURF 保留策略。
  • 你认可后，我就可以按这个规范继续写正式设计文档和分期实现计划。

当前提案里的关键约束

  • `SURF` / `SURF 特征匹配` 规范保留，当前 SDK 落地先用 `SIFT / ORB / AKAZE` 体系占位。
  • 直方图、关键点、匹配结果不再只算“处理后图片”，需要支持附加结果视图。
  • 仿射、透视、GrabCut、浸水填充、特征匹配，必须有额外交互输入能力。
  • `亮度对比度` 和 `|y = kx + b|` 我先按两个可并存算子写，后面你可以决定是否合并。
